## Why Containerize?

Consistent environments from development to production, eliminating the "it works on my machine" problem.

## Multi-stage Builds

Use multi-stage Dockerfiles to keep production images small:

```dockerfile
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM node:20-alpine AS runner
WORKDIR /app
COPY --from=builder /app/.next ./.next
COPY --from=builder /app/node_modules ./node_modules
EXPOSE 3000
CMD ["npm", "start"]
```

## Docker Compose for Development

Set up your entire stack — Next.js, PostgreSQL, Redis — with a single `docker compose up`.

## Tips

- Use `.dockerignore` to avoid sending `node_modules` to the Docker daemon
- Pin base image versions for reproducible builds
- Use health checks for service dependencies
